Shell脚本备份MySQL总结

需求：
备份策略：每天一次增量，每周一次全量
备份方式：至少两种，并且可以通过传参选择
通知策略：备份失败，发送通知
存储方式：按日期为文件名存储，本地和异地备份
回收机制：可配置备份文件保留时间
代码管理：将代码提交到公司git库里
梳理：
1.mysqldump和xtrabackup两种备份方式通过两个脚本来实现
2.选择增量或全量备份方式通过判断日期来决定
3.备份成功，保存异地；备份失败，发送通知邮件
(在这里把邮箱地址写到了一个文件里，可以随便增加邮箱，发邮件的时候读取这个文件，实现可以同时发给多人的功能。)
4.根据传入的时间决定删除哪些老备份
